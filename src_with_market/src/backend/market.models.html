Programmer: Heet
 
Brief:
    Data model for the marketplace domain. Defines Category, Product, SellerProfile,
    Listing, Order, and OrderItem. These models back the REST API and enforce core
    marketplace invariants (e.g., listings reference products and sellers; orders
    are composed of order items).
 
Important data structures:
    - Category: name/slug lookup with unique constraint, used to organize products.
    - Product: abstract thing being sold; belongs to exactly one Category.
    - SellerProfile: 1â€“1 wrapper around Django's User to store seller-facing fields.
    - Listing: a sellable SKU (product + seller + price + stock + active flag).
    - Order: a buyer's purchase; status and monetary total tracked here.
    - OrderItem: a line item in an Order pointing to a specific Listing.
 
Algorithms/design notes:
    - Orders are calculated server-side. Totals are derived by summing OrderItems.
    - Stock is decremented inside a DB transaction during order creation (see serializer),
      which prevents race conditions when multiple buyers purchase simultaneously.
